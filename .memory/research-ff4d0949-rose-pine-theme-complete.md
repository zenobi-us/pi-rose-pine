## Summary

Rose Pine is a comprehensive colour theme project with the tagline "All natural pine, faux fur and a bit of soho vibes for the classy minimalist." It features three main variants (main, moon, and dawn) with 15 carefully curated colours each. The project emphasizes minimalist aesthetics with a cohesive design philosophy across 200+ ports for various applications and platforms.

## Theme Variants

Rose Pine includes **three official variants**:

1. **Rosé Pine (Main)** - The primary dark variant
2. **Rosé Pine Moon** - A deeper, darker variant
3. **Rosé Pine Dawn** - A light variant

### Naming Conventions
- Codenames: `main`, `moon`, `dawn`
- File names use snake-case: `rose-pine`, `rose-pine-moon`, `rose-pine-dawn`
- When referencing, "main" is typically omitted unless necessary for clarity

## Colour Roles & Usage Guidelines

### Neutral Colours

| Role | Description | Usage Examples |
|------|-------------|----------------|
| **base** | Primary background | Inactive tabs, sidebars, application frames |
| **surface** | Low contrast background atop base | Text inputs, panels, cards, status lines |
| **overlay** | Medium contrast background atop surface | Popovers, notifications, dialogs, active tabs |
| **muted** | Low contrast foreground | Comments, git ignored files, disabled elements, unfocused text |
| **subtle** | Medium contrast foreground | Non-selected results, inactive tabs, punctuation, operators, secondary content |
| **text** | High contrast foreground | Cursor text, selected results, active tabs, variables, normal text, selection foreground (paired with highlightMed background) |
| **highlightLow** | Low contrast highlight | Cursor line background |
| **highlightMed** | Medium contrast highlight | Selection background (paired with text foreground) |
| **highlightHigh** | High contrast highlight | Cursor background, borders, visual dividers |

### Accent Colours

| Role | Terminal ANSI | Usage Examples |
|------|---------------|----------------|
| **love** | Red, Bright Red | Terminal red, builtins, diagnostic errors, deleted Git files |
| **gold** | Yellow, Bright Yellow | Terminal yellow, strings, diagnostic warnings |
| **rose** | Cyan, Bright Cyan | Terminal cyan, booleans, functions, matching search background, modified Git files, git text |
| **pine** | Green, Bright Green | Terminal green, functions, conditionals, keywords, renamed Git files |
| **foam** | Blue, Bright Blue | Terminal blue, object keys, tags, types, diagnostic information, Git additions |
| **iris** | Magenta, Bright Magenta | Terminal magenta, methods, parameters, diagnostic hints, inline links, merged and staged Git modifications |

## Design Philosophy

### Core Principles

1. **Minimalist Aesthetic**
   - "All natural pine, faux fur and a bit of soho vibes for the classy minimalist"
   - Clean, uncluttered design
   - Carefully curated colour selection (15 colours per variant)

2. **Natural & Organic**
   - Inspired by natural elements (pine, rose, foam)
   - Soft, muted tones that are easy on the eyes
   - Cohesive colour harmony across all variants

3. **Accessibility**
   - Three variants to suit different preferences and lighting conditions
   - Clear contrast ratios for readability
   - Consistent colour roles across variants

4. **Consistency**
   - Unified design language across 200+ ports
   - Standardized colour roles and usage
   - Coherent implementation guidelines

### Design Inspiration

The theme draws inspiration from:
- **Natural elements**: Pine trees, rose flowers, sea foam
- **Soho aesthetics**: Urban sophistication, artistic sensibility
- **Minimalism**: "Classy minimalist" approach
- **Comfort**: "Faux fur" suggests softness and comfort

- **Current Version:** v4.0.1
- **Package:** @rose-pine/palette on npm

#### Features

1. **Multiple Export Formats**
   - CSS variables (media-based, class-based, static)
   - Tailwind CSS presets
   - JavaScript/TypeScript imports

2. **Colour Formats**
   - HEX (without # prefix in v4+)
   - RGB (array format)
   - HSL (array format)
   - Alpha variants for highlights

3. **Data Structure** (v4.0.1+)
   ```javascript
   // Variants
   variants.main.colors.rose.hex // => 'ebbcba'
   variants.moon.colors.rose.hex // => 'ea9a97'
   variants.dawn.colors.rose.hex // => 'd7827e'
   
   // Roles
   roles.base.colors.main.hex // => '191724'
   roles.base.colors.moon.hex // => '232136'
   roles.base.colors.dawn.hex // => 'faf4ed'
   ```
